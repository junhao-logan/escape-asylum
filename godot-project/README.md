# Godot 工程

> 这个文件夹放 Godot 工程本体。**由主程负责创建**，因为 Godot 版本、
> 语言选择（GDScript / C#）、渲染设置等需要程序团队统一决定。

## 创建步骤（主程操作）

1. 团队统一一个 Godot 版本（建议用最新稳定版的 Godot 4）。
2. 用 Godot 新建工程，工程直接建在本文件夹内（会生成 `project.godot`）。
3. 确认根目录的 `.gitignore` 对 Godot 生效（`.godot/` 缓存目录已配置忽略）。
4. 第一次提交前，跑 `git status` 确认 `.godot/` 没有被误提交。
5. 是否启用 Git LFS 见下方说明。

## 关于语言：GDScript 还是 C#

- **GDScript**：Godot 自带语言，和引擎集成最好、上手快、迭代快。**推荐团队默认用它。**
- **C#**：如果团队来自 Unity、想复用 C# 经验也可以，但配置略麻烦、部分平台导出有额外要求。
- 建议：先用 GDScript 把 Slice 做出来，不要在语言选择上消耗时间。

> 团队最终决定：_____（GDScript / C#）

## 关于 Git LFS

Godot 工程通常比 Unity 轻。**早期美术资源不多时，可以先不启用 LFS**，
少一个出错环节。等高清贴图、音频、视频等大文件多起来了再启用：
根目录 `.gitattributes` 已经配好规则，到时在仓库里跑 `git lfs install` 即可。

## 建议的工程内目录结构

在工程根目录下建立清晰分类（供参考，团队可调整）：

```
godot-project/
├── project.godot        ← Godot 工程文件
├── scenes/              ← 场景：战斗、房间、UI、主菜单
├── scripts/             ← 脚本：战斗、卡牌、关卡、数据
├── art/                 ← 美术：角色、场景、UI、特效
├── audio/               ← 音频
├── data/                ← 卡牌表、敌人表等数据资源（.tres / .json）
└── addons/              ← 第三方插件
```

## 协作注意（Godot 专属）

- Godot 的 `.tscn` 场景文件是文本格式，**两个人同时改同一个场景，合并时容易冲突**。
  约定：尽量避免多人同时改同一场景；需要协作时提前沟通、错开。
- 美术的概念图、参考图放在 `docs/美术/`，**进了游戏的正式资源**才放这里。
- 提交前检查 `git status`，别把 `.godot/` 临时文件提交进仓库。
