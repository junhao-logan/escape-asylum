# 如何把项目上下文用到自己的 Claude Project

> 本文件夹是专门为团队成员**各自的 Claude Project** 准备的。

## 为什么需要这个文件夹

整个仓库的文档很多、很碎，而且不少还在 work-in-progress。
如果直接把整个 `docs/` 上传到 Claude Project，Claude 会被无关信息干扰。

所以这里放了一份**精简、自洽、随时最新**的项目上下文文件，
任何人把它丢进自己的 Claude Project，Claude 就能立刻理解整个游戏。

## 怎么用

1. 在 Claude.ai 创建一个 Project（例如"潜逃收容所-策划"或"潜逃收容所-程序"）。
2. 把本文件夹里的 `项目上下文_精简版.md` 上传为 Project 知识 / 文档。
3. 之后在这个 Project 里和 Claude 对话，它都会带着项目背景。
4. 需要更细的内容时，再单独上传相关的具体文档（比如某个 GDD 分篇、某张数据表）。

## 维护约定

- `项目上下文_精简版.md` 是**唯一需要保持最新**的上下文文件。
- 当游戏方向有重大变化时，更新它，并通知团队重新上传。
- 不要把它写得太长——它的价值就在于"精简"。控制在两三屏以内。

## 各角色建议的 Project 配置

| 角色 | Project 里建议放的文档 |
|---|---|
| 主策划 | 项目上下文 + 全部 GDD + 数据表 |
| 程序 | 项目上下文 + 战斗系统篇 + 卡牌系统篇 + 数据表 |
| 美术 | 项目上下文 + 美术规范 + 世界观篇 |
| 宣发 | 项目上下文 + pitch 文件夹全部 |
