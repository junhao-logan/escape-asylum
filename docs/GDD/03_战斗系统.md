# 03 · 战斗系统

> ⚠️ 这是目前**最需要先敲定**的系统。程序团队动手前必须先把本篇的待定问题解决。

## 已知设定

- 战斗视角：第三人称。
- 玩家可释放**时缓（时停）**，在时停中从容打牌。
- 非时停状态下可用**冲刺**躲避攻击。
- 能量条**被动回复**，卡牌消耗能量（见数据表）。
- 技能释放需用鼠标选方向，可用鼠标左右键取消施法。
- 敌人大多是"机制怪"。

## 需要敲定的核心问题（拦路问题）

以下问题不解决，战斗系统无法进入开发：

- [ ] **时缓如何触发与结束**：主动按键？有冷却 / 消耗？持续多久？
- [ ] **时停中的出牌规则**：能出几张？手牌打完才刷新弃牌堆 —— 那时停就结束吗？
- [ ] **能量恢复速率**：每秒回多少？上限多少？被哪些因素影响？
- [ ] **冲刺与卡牌的配合**：冲刺有无消耗 / 冷却？时停时能否冲刺？
- [ ] **"机制怪"具体指什么**：列出 MVP 阶段的 2–3 种敌人机制原型。
- [ ] **伤害与受击**：玩家血量、受击反馈、死亡判定。
- [ ] **施法方向选择的手感**：瞄准如何呈现？

## MVP 阶段建议范围

为了尽快验证"时停打牌爽不爽"，第一个可玩版本建议只做：

- 1 个主角能力（流派）
- 时缓 + 能量 + 冲刺 的完整闭环
- 2–3 种敌人机制 + 1 个 BOSS
- 15–20 张卡覆盖攻击 / 防御 / buff 基本类型
