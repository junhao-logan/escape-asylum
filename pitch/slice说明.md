# Vertical Slice 说明

> Slice 是给投资人 / 发行商看的"切片演示"。它不是完整游戏，但要让人**玩到**
> 这游戏最核心、最好玩的那段体验。本文件定义 Slice 要做成什么样。

## Slice 的目的

让投资人在 10–15 分钟内亲身体验并相信：
1. 核心玩法（时停打牌 + 七宗罪构筑）是**好玩的**。
2. 团队有能力把它**做出来并做精**。
3. 美术与氛围**有辨识度、有市场卖相**。

## Slice 应包含的内容

| 模块 | 范围 | 说明 |
|---|---|---|
| 核心循环 | 完整一遍 | 进房间 → 战斗 → 拿卡 → 强化 → 下一间 |
| 流派 | 1 个 | 七宗罪选一个最有展示力的 |
| 关卡 | 第一层 | 含数种房间类型 + 1 个 BOSS |
| 卡牌 | 15–20 张 | 覆盖攻击/防御/buff，能体现构筑 |
| 敌人 | 2–3 种 + 1 BOSS | 体现"机制怪"设计 |
| 美术 | 正式美术 | 这一层不能用占位素材，要做出卖相 |
| 叙事 | 一段切片 | 开场氛围 + 流派/世界观钩子 |
| 打磨 | 高 | 手感、音效、正反馈——Slice 的成败在打磨 |

## Slice 不需要包含

- 全部 7 层、全部 7 个流派、113 张卡——这些是完整版内容。
- 完整剧情、真假结局。
- 复杂的局外成长系统（可点到为止）。

## 验收标准

- [ ] 能从头玩到尾、不崩溃。
- [ ] 没玩过的人不需要讲解也能上手。
- [ ] 试玩者结束后"还想再玩一局"。
- [ ] 有一份配套的 Demo 录像（给无法到场的投资人）。

## 风险提示

Slice 最常见的失败是**摊太大**——什么都做一点，结果什么都不精。
宁可范围小，也要让玩到的那一段足够惊艳。范围一旦定下，严格冻结。
